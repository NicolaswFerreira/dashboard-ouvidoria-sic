# Metodologia

## 1. Base de dados

A base utilizada neste projeto é simulada e foi estruturada para representar um fluxo administrativo de manifestações de Ouvidoria/SIC.

Foram considerados campos como:

- data de entrada;
- data de resposta;
- status;
- unidade responsável;
- tipo de manifestação;
- assunto;
- prazo;
- situação da resposta.

## 2. Tratamento dos dados

Foram realizadas etapas de:

- padronização de nomes de unidades;
- classificação de status;
- cálculo de prazo de resposta;
- identificação de respostas dentro e fora do prazo;
- agrupamento por mês;
- preparação da base para tabelas dinâmicas.

## 3. Indicadores

Os indicadores foram definidos com foco em acompanhamento gerencial, permitindo visualizar volume de demandas, cumprimento de prazos e possíveis gargalos por unidade.

## 4. Limitações

Por se tratar de base simulada, os dados não representam números oficiais. O objetivo do projeto é demonstrar a estrutura analítica e a lógica de construção do dashboard.
